# Run Portal Frontend Brief

## Page

`run-portal.html`

## Visual Direction

An operator cockpit with editorial hierarchy:

- calm, bright background
- strong top decision block
- restrained blue/ink palette
- dense information, but only after progressive disclosure

## Typography

- serif or humanist heading tone for section titles
- compact, neutral body text
- keep metrics and modes in monospace chips

## Component Rules

### Hero

- should look like a workspace header, not a generic card
- include a metric strip with strong separation between metadata and action

### Decision Card

- most visually prominent block on the page
- distinct background treatment from supporting cards

### Capability Chips

- compact pill treatment
- visually distinct for available vs missing capability groups

### Entry Cards

- primary cards must feel action-oriented
- secondary/reference cards must visually step down
- each card should show:
  - title
  - purpose
  - key status
  - recommended entry point
  - linked objects summary

### Folded Blocks

- folded summaries must still read as useful summaries, not hidden junk drawers
- the summary line should explain what is being hidden

## Interaction Rules

- keep motion subtle
- highlight primary actions, not every link
- do not overload with decorative effects

## Large-Data Behavior

- strong visual separation between first-read content and secondary catalogs
- first screen must never look like a link farm
- summary counts and short reasons must dominate over raw lists

## Do Not Change

- semantic labels and JSON-derived terminology
- object links and report anchors
