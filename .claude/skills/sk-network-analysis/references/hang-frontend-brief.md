# Hang / Crash Frontend Brief

## 视觉目标

把 `hang-crash-report.html` 做成“故障分诊页”，不是告警目录页。

## 视觉原则

- 首屏只有一个主判断卡
- 主信号表格用主卡片承载
- 阶段关联和 Queue / DFX 走次级卡片
- 次级明细默认折叠

## 组件约束

- 主判断使用高对比 priority box
- 主信号表格表头和内容全部中文
- 折叠区标题要短，不要长句解释
- 页面内不要出现大段说明文字

## 风格约束

- 与 `run-portal` 保持同一套卡片语言
- 不新增复杂图形，仅用简洁卡片、表格、折叠区完成
