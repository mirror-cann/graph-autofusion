# Performance IA Brief

## 页面

`performance-report.html`

## 页面目标

把性能页收成事件主视角的客户主页面，先回答：

- 当前性能问题主要落在哪个事件分组
- 该先跳哪个 scope / task / update 对象
- 当前矩阵里哪几行值得先看

## 首要用户任务

用户应在 15 秒内知道：

- 当前焦点是什么
- 哪个事件分组最值得先看
- 下一步该跳哪里验证

## 默认信息层级

1. `当前焦点`
2. `事件主视角矩阵`
3. `热点事件`
4. `关联入口`
5. `更多矩阵行`

## 大数据默认策略

- 事件主矩阵放首屏
- 热点事件放次级区
- 更多矩阵行默认折叠
- 不再让 update/function 成为页面主叙事

## 不允许改动

- `sk_event_dev_device_*.json` 作为性能主资产的定位
- 现有 `recommended_entry` 语义
- query / 锚点 / 文件名
